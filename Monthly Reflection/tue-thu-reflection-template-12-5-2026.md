**Why did you build it THIS way?** 
We found the options to build this system in several ways. The first option is to create a rule-based framework, where each complaint is checked against predefined behavioral and cognitive indicators such as expectation gap, perceived unfairness, administrative burden, blame attribution, emotional pressure, and trust level. The second option is to use a manual coding method, where researchers read a sample of complaints, label the main behavioral patterns, and then build clear classification rules from these findings. The third option is to use a persona-mapping framework, where each complaint is connected to a citizen persona based on the issue, emotional driver, complaint style, and repeated pain points. The fourth option is to combine all of these into a hybrid model, where the system first identifies the issue, then analyzes the behavioral signals, and finally maps the citizen to the most suitable persona with human review for sensitive or unclear cases.
