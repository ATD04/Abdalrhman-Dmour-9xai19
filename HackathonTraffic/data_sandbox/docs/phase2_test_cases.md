# Phase 2: Technical Test Cases

The following test cases define the verification protocol for the Phase 2 Crack-the-Code build.

## TC-01: Modular Ingestion Integrity
- **Objective:** Verify that the Data Acquisition Layer correctly normalizes heterogeneous CSV sources.
- **Procedure:** Run `phase2_data_acquisition.py`. Check `normalized_demand.json`.
- **Expected:** ISO 8601 timestamps, consistent column names, no field mismatches.
- **Status:** ✅ VERIFIED

## TC-02: Schema-Compliant Notification
- **Objective:** Ensure incident detection outputs valid JSON matching `event_notification_schema.json`.
- **Procedure:** Inspect `event_notifications.json` generated by `phase2_incident_detection.py`.
- **Expected:** Contains `event_id`, `confidence_score`, and `severity`. No missing mandatory fields.
- **Status:** ✅ VERIFIED

## TC-03: Forecasting Baseline Comparison
- **Objective:** Prove that the forecasting model provides a non-trivial improvement over simple baselines.
- **Procedure:** Run `phase2_forecasting.py`. Review `forecasting_benchmarks.json`.
- **Expected:** Final model MAE < Naive Baseline MAE.
- **Status:** ✅ VERIFIED (15.94 < 18.45)

## TC-04: Decision Support Isolation
- **Objective:** Confirm that Signal Optimization recommendations are read-only and non-operational.
- **Procedure:** Inspect `signal_recommendations.json`. Check for any "execute" or "write" commands to signal controllers.
- **Expected:** Status field must be "Decision Support Only". Recommendations contain only suggested changes in seconds.
- **Status:** ✅ VERIFIED

## TC-05: System Health HUD Synchronization
- **Objective:** Verify that the Dashboard correctly surfaces backend health indicators.
- **Procedure:** View "System Health" panel in React Dashboard. Compare with `/api/v1/phase2/system-health` response.
- **Expected:** Real-time display of ingestion status and invalid record count.
- **Status:** ✅ VERIFIED

## TC-06: Dual-Source Synchronization
- **Objective:** Ensure CV overlays remain frame-synced with the video playback.
- **Procedure:** Play video in dashboard. Observe bounding boxes during vehicle movement.
- **Expected:** No visible lag between vehicle position and bounding box. Count increments only when vehicles cross trip-lines.
- **Status:** ✅ VERIFIED
