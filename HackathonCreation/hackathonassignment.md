■ Hackathon Assignment: Build a
Hackathon-in-a-Box
Overview
You are not participating in a hackathon.
Your challenge is to build a tool that creates and runs a hackathon for others.
Design and develop a platform that allows any organizer to generate a complete hackathon
experience in minutes.
Problem Statement
Build a web application that can automatically generate a full hackathon plan based on inputs like
participants, goals, and constraints.
Your Goal
Create an MVP called Hackathon-in-a-Box — a tool that transforms inputs into a ready-to-run
hackathon.
Core Features (Required)
1. Input Form
1 Number of participants
2 Skill levels
3 Duration
4 Domain
5 Team size
6 Learning goals
2. Hackathon Generator
1 Hackathon name & theme
2 Description
3 Challenge tracks
4 Rules & guidelines
5 Timeline
6 Judging criteria
3. Team Generator
Automatically divide participants into fair teams.
4. Idea Prompt Generator
Generate starter ideas for teams.
5. Export / Share
Output a clean, shareable hackathon brief.
Bonus Features
1 AI-generated ideas
2 Leaderboard
3 Judge scoring
4 Certificates
5 Surprise constraints
Deliverables
1 Working prototype
2 Demo
3 Architecture overview
4 Example output
5 Design decisions
Evaluation Criteria
1 Usefulness
2 UX
3 Output quality
4 Technical implementation
5 Creativity
Constraints
Work in teams and focus on a strong MVP.
Final Note
Build something real that you can showcase and use.
One-Line Summary
Build a platform that can design and run a hackathon for someone else.
